# META 规则汇总

> 从项目错误中提炼的跨项目通用规则。新项目启动时读取此文件作为预置防线。

## 格式规范

```markdown
### META-XXX: 规则标题

- **规则**: 通用规则描述
- **适用场景**: 哪类代码/场景需要遵守
- **源错误**: ERR-NNN（可跨项目引用）
- **检查方式**: 如何在代码审查中验证
```

---

| 编号 | 规则 | 适用场景 | 源错误 |
|------|------|---------|--------|
| （无） | | | |

---

## 使用方式

1. **新项目启动**：读取此文件，将 META 规则作为代码审查检查项
2. **新错误发生**：按五步法记录到 `errors/raw/ERR-NNN.md`，提炼完成后在此追加 META 规则
3. **复现检查**：若某 ERR 复现次数 >1，说明提炼不够，需回溯 raw 记录重新分析
4. **跨项目传播**：将此文件复制到新项目的 `errors/distilled/` 即可继承防线
