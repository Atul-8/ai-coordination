# ERR-000: 异步函数未做错误处理导致数据丢失

> 此文件是五步法示例记录，展示完整的错误分析过程。实际使用时按此格式填写。

- **症状**: 用户提交表单后，页面显示"保存成功"但数据库中无记录。控制台无报错信息，网络请求返回 200。
- **根因**:
  1. 为什么数据没入库？→ async saveData() 抛出了数据库连接超时异常，但未被捕获
  2. 为什么未捕获？→ 调用方 `handleSubmit()` 使用了 `await saveData()` 但没有 try-catch
  3. 为什么没 try-catch 成了惯例？→ 项目中所有 async 函数都默认"不会出错"，没有强制错误处理的编码规范
- **修复**: 在 `handleSubmit()` 中添加 try-catch，失败时向用户展示错误提示而非虚假的"保存成功"
- **规律提炼**: 所有 async 函数都可能抛出异常，调用方必须处理异常情况不可假装不会出错
- **二次提炼接口**: META-ASYNC — 所有 async 函数调用处必须包含 try-catch 或等效错误处理（如 .catch()），禁止裸 await 不含错误处理

- **复现次数**: 1
