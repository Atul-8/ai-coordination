# 错误知识库

> 累积调试错误记录，分析根因，沉淀为可复用知识。
> 每条记录遵循「错误 → 根因 → 修复 → 规律提炼 → 二次提炼接口」五步法。

## 格式规范

```markdown
### ERR-NNN: 错误标题

- **症状**: 现象描述
- **根因**: 深层原因分析（不是表面原因）
- **修复**: 具体修复方式
- **规律提炼**: 从此错误抽象出的通用规则
- **二次提炼接口**: 可跨项目复用的检查项/编码规范
- **首次出现**: YYYY-MM-DD
- **复现次数**: N（若 >1 说明规律提炼不够透彻）
```

### 使用方式

1. **新项目启动**：读取此文件，将 META-xxx 规则作为代码审查检查项
2. **新错误发生**：按五步法记录，提炼新的 META 规则
3. **复现检查**：若某 ERR 复现次数 >1，说明规律提炼不够，需重新分析根因
4. **定期回溯**：每次会话开始时扫描复现次数，优先加固高频错误
