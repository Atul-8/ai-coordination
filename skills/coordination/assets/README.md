# .ai/ 对接层目录

> 本目录是 ai-coordination 插件的工作区，承载项目的开发状态持久化、上下文容灾和错误知识管理。

## 文件说明

| 文件 | 更新方式 | 说明 |
|------|---------|------|
| `WORKSTATE.md` | 覆盖 | 当前工作状态、未完成任务、上次中断点。新会话**最高优先**读取此文件 |
| `STRUCTURE.md` | 覆盖 | 代码结构地图，架构变更时同步更新 |

## 文件夹说明

| 目录 | 更新方式 | 说明 |
|------|---------|------|
| `changelog/` | 追加 | 操作履历。每条记录一次关键操作，按日期归档 |
| `requirements/` | 追加 | 需求记录。按 REQ-NNN 编号管理，追踪状态和变更影响 |
| `errors/` | 追加 | 错误知识库，分为原始记录和提炼成果 |

### errors/ 目录详解

```
errors/
  raw/              # 原始错误记录（五步法完整过程）
    ERR-001.md       # 每条错误独立文件
    ERR-002.md
    ...
  distilled/        # 提炼成果（可跨项目共享）
    meta-rules.md    # META 规则汇总表
```

**raw/** — 记录错误的完整五步法分析：症状 → 根因 → 修复 → 规律提炼 → 二次提炼接口。属于项目内部资料，通常不对外共享。

**distilled/** — 从原始错误中提炼出的 META 规则汇总。这是"错题本"，适合团队共享和新项目继承：
- 新项目启动时，只需读取 `distilled/meta-rules.md` 即可获得跨项目防线
- 共享仓库时，可以选择只同步 `distilled/`，不暴露原始调试细节

## 共享策略

`.ai/` 目录通过 `/ai:sync` 同步到云端 Git 仓库。不同目录可根据需要选择性共享：

| 目录 | 共享建议 | 理由 |
|------|---------|------|
| `WORKSTATE.md` | 共享 | 多机续接需要 |
| `STRUCTURE.md` | 共享 | 架构共识 |
| `changelog/` | 可选 | 操作历程，对他人参考价值有限 |
| `requirements/` | 共享 | 需求追踪需要团队可见 |
| `errors/raw/` | 不建议 | 包含调试细节，可能涉密 |
| `errors/distilled/` | **强烈建议** | 提炼后的通用规则，最适合跨项目/跨团队传播 |
